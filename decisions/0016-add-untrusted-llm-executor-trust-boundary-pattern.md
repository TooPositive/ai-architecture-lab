# ADR-0016: Add Untrusted-LLM Executor Trust Boundary Pattern

**Date**: 2026-04-08
**Status**: Accepted
**Pipeline**: pattern-lab

## Context

Source describes the RUX orchestration approach: treat the LLM as untrusted and introduce an Executor as a deterministic trust boundary enforcing a schema/contract before any tool invocation, to prevent silent failures from hallucinated actions and to enable reliability measurement.

## Decision

Add Untrusted-LLM Executor Trust Boundary Pattern

## Consequences

This pattern/content has been added to the architecture lab repository.
